Log CSV Gather 사용자 안내
==========================

이 프로그램은 현장 PC의 CSV 로그를 Google Drive로 업로드하거나, 관리 PC에서 Google Drive의 CSV 로그를 내려받기 위한 로컬 전용 웹앱입니다.


1. 처음 실행
-----------

1) 이 폴더의 run.bat을 더블클릭합니다.

2) 처음 실행 시 역할 선택이 나오면 이 PC의 용도를 선택합니다.
   - 1 = 현장 PC 업로드
   - 2 = 관리 PC 다운로드

3) 브라우저가 자동으로 열리면 대시보드의 초기설정 버튼을 누릅니다.

4) 초기설정 창에서 필요한 값을 입력하고 저장합니다.
   - PC 이름: 이 PC를 구분할 이름입니다. 예: field-pc-01, manager-pc-01
   - 역할: 현장 PC 업로드 또는 관리 PC 다운로드
   - Google Drive 루트 폴더 ID: 공유받은 log-csv-gather-root 폴더의 ID입니다.
   - 검사기명: 현장 PC 업로드 역할일 때 필요합니다. 예: 성능검사기_1
   - 로그 폴더 위치: 현장 PC 업로드 역할일 때 필요합니다. 예: E:\
   - 다운로드 위치: 관리 PC 다운로드 역할일 때 필요합니다.

5) 설정 저장 후 Doctor는 자동 실행되지 않습니다. 반드시 아래 순서대로 직접 버튼을 눌러 확인합니다.


2. 권장 테스트 순서
------------------

처음 설정하거나 설정을 바꾼 뒤에는 아래 순서로 진행합니다.

1) Auth
   - Google 로그인 인증을 진행합니다.
   - Google 아이디/비밀번호를 이 프로그램 화면에 입력하지 않습니다.
   - 인증이 끝나면 token.json은 runtime 폴더 아래에 자동 저장됩니다.

2) Doctor
   - Drive 연결, 설정값, 로컬 폴더 준비 상태를 점검합니다.
   - 초록색이면 정상입니다.
   - 노란색이면 동작은 가능하지만 확인이 필요한 항목이 있습니다.
   - 붉은색이면 해당 메시지를 확인하고 수정해야 합니다.

3) Dry-run
   - 실제 업로드/다운로드 없이 대상 파일만 미리 확인합니다.
   - 현장 PC는 업로드 대상 CSV가 몇 개인지 확인합니다.
   - 관리 PC는 다운로드 대상 CSV가 몇 개인지 확인합니다.

4) Once
   - 실제 업로드 또는 다운로드를 1회 실행합니다.
   - Current Job의 진행률과 Job Feed의 결과를 확인합니다.

5) Scheduler
   - Once 테스트가 정상일 때만 등록/갱신합니다.
   - 기본 주기는 60분입니다.
   - 대시보드에서 주기를 바꾸거나 등록/해제할 수 있습니다.


3. 현장 PC 업로드 규칙
---------------------

업로드 대상은 로그 폴더 아래의 다음 4개 폴더입니다.

- PAS Test data
- HM-3203-011 Test data
- HM-3903-011 Test data
- LITE Test data

다음 항목은 무시됩니다.

- fail 폴더
- 등록되지 않은 다른 폴더
- 날짜 형식이 YYYYMMDD가 아닌 폴더
- 파일명에 总数据가 없는 CSV
- 확장자가 .csv가 아닌 파일
- 마지막 수정 후 안정화 시간이 지나지 않은 파일

Drive에는 다음 규칙으로 저장됩니다.

logs/Array_MIC/{PAS|3203|3903|LITE}/{검사기명}/YYMMDD/YYMMDD_{type}.csv


4. 화면 보는 법
---------------

- Action 버튼 상태
  - 초록색: 성공
  - 노란색: 경고 또는 확인 필요
  - 붉은색: 실패

- Current Job
  - 현재 실행 중인 작업, 성공/실패 아이콘, 진행률을 표시합니다.

- Job Feed
  - 최신 이벤트가 위쪽에 표시됩니다.
  - 실패가 발생하면 메시지를 확인한 뒤 필요하면 다시 실행합니다.

- App Log
  - 자세한 오류 확인용입니다.
  - 일반 작업자는 Doctor, Current Job, Job Feed를 먼저 확인하면 됩니다.


5. 실패와 재시도
----------------

인터넷이 잠시 불안정하거나 Drive 요청이 일시적으로 실패하면, 프로그램은 해당 파일을 기록해두고 나머지 파일을 먼저 처리합니다.

작업 마지막에 실패한 파일만 한 번 더 재시도합니다.

다만 다음 상황은 자동 재시도하지 않습니다.

- 로컬 로그 폴더가 없음
- 다운로드 위치가 파일로 잡혀 있음
- Drive 권한이 없음
- 이미 있는 파일과 내용이 달라 conflict가 발생함

이 경우 설정, 권한, 경로를 수정한 뒤 Doctor와 Dry-run부터 다시 확인합니다.


6. 역할 또는 설정을 잘못 선택한 경우
------------------------------------

대시보드에서 초기설정 버튼을 다시 눌러 값을 수정합니다.

역할 자체를 바꿔야 할 때도 초기설정 창에서 현장 PC 업로드 또는 관리 PC 다운로드를 다시 선택할 수 있습니다.

active.yaml 초기화 기능은 다음 실행 때 역할 선택부터 다시 시작하고 싶을 때 사용합니다.

스케줄러가 등록된 상태에서 역할 또는 주요 설정을 바꾸면, 안전을 위해 기존 스케줄러 등록은 해제될 수 있습니다. 설정 저장 후 Doctor, Dry-run, Once를 다시 확인하고 Scheduler를 다시 등록합니다.


7. 주의사항
-----------

- Google 계정 비밀번호를 프로그램, 설정 파일, 메모장 등에 저장하지 않습니다.
- token.json은 PC별 인증 파일입니다. 다른 PC에 복사해 쓰는 방식은 권장하지 않습니다.
- secrets 폴더와 runtime 폴더는 삭제하지 않는 것이 좋습니다.
- 테스트 전에는 Scheduler보다 Doctor, Dry-run, Once를 먼저 사용합니다.
- 이 웹앱은 로컬 전용이며 기본 주소는 http://127.0.0.1:8765 입니다.
